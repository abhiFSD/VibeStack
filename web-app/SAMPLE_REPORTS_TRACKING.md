## Sample Data Structure

For each report type, we create 3 sample files with different characteristics:

### **File Naming Convention**
```
{report_type}_report_1.json  // High Performance Scenario
{report_type}_report_2.json  // Average Performance Scenario  
{report_type}_report_3.json  // Needs Improvement Scenario
```

### **Sample Scenarios**

#### **Scenario 1: High Performance**
- **Status**: Completed
- **Scores**: Mostly 4-5 range
- **Team Engagement**: High (multiple assignees)
- **Use Case**: Showcase best practices

#### **Scenario 2: Average Performance**
- **Status**: In Progress
- **Scores**: Mixed 2-4 range
- **Team Engagement**: Moderate
- **Use Case**: Typical real-world assessment

#### **Scenario 3: Needs Improvement**
- **Status**: In Progress or Completed with issues
- **Scores**: Lower 1-3 range
- **Team Engagement**: Minimal
- **Use Case**: Areas requiring attention
